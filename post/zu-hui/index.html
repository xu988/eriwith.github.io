MVVM

Model - View - ViewModel

为什么使用MVVM

  • 团队 ：
    • 确定一个框架 便于维护（其他框架也一样）
    • 新人无需很大的成本去了解原来的逻辑
  • 架构 ：
    • 耦合度低（view不包含业务逻辑）
    • 稳定
  • 代码 ：
    • 可读性强（只要遵循了mvvm框架开发，找每一个代码的位置就会很方便）
    • 可测性强（业务逻辑和界面逻辑可以分别测试）
    • 可替换（更新新的实现可以不影响其他部分）

什么是Model

wpf是模型驱动开发

数据是核心 ui由内部数据来驱动 ui将操作和数据传回内部的数据

  • Model是现实世界的抽象结果

什么是View和ViewModel

  • View = UI （View是对Model的展现 客户观察数据的方式）
  • ViewModel 是为了响应View的操作 （Model for View，是对View的一种抽象）
  • Model 与 View 的沟通：
    • 数据传递 通过 数据属性
    • 操作传递 通过 命令属性
